# recipe-box-api

Минимальный backend для RecipeBox.

## Каркас

```txt
src/
├── server.js
├── app.js
├── config/
├── middlewares/
├── routes/
├── modules/
└── utils/
```

## Назначение

`recipe-box-api` отвечает за серверную часть RecipeBox:

- аутентификацию и авторизацию на JWT
- работу с пользователями
- работу с рецептами
- работу с планами питания
- единый HTTP API для фронтенда

## Слои каркаса

### `src/server.js`

Точка входа приложения. Отвечает за запуск HTTP-сервера и соединение с окружением.

### `src/app.js`

Собирает Express-приложение:

- подключает middleware
- настраивает CORS
- монтирует маршруты API
- подключает обработку 404 и ошибок

### `src/config/`

Конфигурационный слой:

- чтение переменных окружения
- настройки порта
- настройки MongoDB
- настройки JWT
- настройки CORS-origin

### `src/middlewares/`

Общий слой промежуточной обработки:

- обработка 404
- обработка ошибок
- защита маршрутов
- валидация входных данных

### `src/routes/`

Слой маршрутов верхнего уровня:

- `/health`
- `/api/v1/auth`
- `/api/v1/users`
- `/api/v1/recipes`
- `/api/v1/meal-plans`

### `src/modules/`

Доменные модули приложения:

- `auth/`
- `users/`
- `recipes/`
- `meal-plans/`

Каждый модуль должен содержать только свой доменный код и не тянуть чужие обязанности.

### `src/utils/`

Общие утилиты, не привязанные к конкретному домену.

## Контракт окружения

Backend использует переменные окружения:

- `PORT`
- `MONGODB_URI`
- `JWT_SECRET`
- `CLIENT_ORIGIN`

## API-подход

API строится как REST-сервис с единым префиксом `/api/v1`.

## Запуск

```bash
npm install
npm run dev
```
